Proyecto

Sistema de Gestión Comercial para Comercios de Productos de Limpieza.

El sistema funcionará como SaaS Multi-Tenant.

Cada comercio tendrá acceso únicamente a sus propios datos.

Componentes

Base de Datos

Todas las entidades deben utilizar:

UUID como clave primaria
createdAt
updatedAt

Nunca utilizar IDs autoincrementales.

Multi Tenant

Este sistema es multi tenant.

Cada consulta a la base de datos debe filtrar obligatoriamente por:

tenantId

o

comercioId

según corresponda.

Nunca permitir acceso cruzado entre comercios.

La seguridad multi tenant es obligatoria.

Gestión de Stock

El stock nunca debe modificarse directamente.

Toda modificación debe generar un MovimientoStock.

Tipos:

INGRESO
VENTA
AJUSTE_POSITIVO
AJUSTE_NEGATIVO

El stock actual será la suma de movimientos.

Puede mantenerse un campo stockActual optimizado para consultas rápidas.
